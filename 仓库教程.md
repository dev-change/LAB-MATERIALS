## 二、快速下载（不用下整个仓库）

很多时候你只需要仓库里的**某一个文件**或**某一个文件夹**，不用把整个仓库克隆下来。

### 方法 1：网页直接下载（最简单）

1. 打开你想下载的文件
2. 点右上角的 **Raw** 按钮
3. 在打开的纯文本页面，按 `Ctrl + S`（或右键 → 另存为）即可保存

> 适合：单个 .md、.pdf、代码文件等

### 方法 2：下载整个文件夹（用 DownGit）

1. 打开 https://downgit.github.io/
2. 复制 GitHub 上你想下载的文件夹链接，粘贴进去
3. 点 **Download**，自动打包成 zip 下载

> 适合：某个项目文件夹、某类资料合集

### 方法 3：克隆整个仓库（需要 Git）

如果你确实需要整个仓库：

```bash
git clone https://github.com/dev-change/Embedded-Main.git
```

> 适合：维护人、频繁提交资料的同学

## 三、Git 快速上手

如果你打算经常提交资料，建议学一下 Git。这里只教最常用的几条命令。

### 安装 Git

- Windows：https://git-scm.com/download/win
- Mac：`brew install git`
- Linux：`sudo apt install git`

### 配置（只需一次）

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 常用命令

```bash
# 克隆仓库到本地
git clone https://github.com/dev-change/Embedded-Main.git

# 进入仓库目录
cd Embedded-Main

# 查看当前修改了哪些文件
git status

# 把修改的文件加入待提交列表
git add .

# 提交（写一句说明这次改了什么）
git commit -m "add: 新增了xxx资料"

# 推送到 GitHub
git push origin main
```

> 更多问题请自行搜索，或直接在群里问。

## 四、如何提交资料

有四种方式，按你的熟练程度任选一种。

---

### 方式 A：Fork + Pull Request（推荐，会 Git 的同学）

这是最标准的开源协作流程：

1. **Fork 仓库**：打开 https://github.com/dev-change/Embedded-Main ，点右上角 **Fork** 按钮，创建一个你自己的副本
2. **克隆你的 Fork**：
   ```bash
   git clone https://github.com/你的用户名/Embedded-Main.git
   cd Embedded-Main
   ```
3. **修改或新增文件**，放到正确的文件夹里
4. **提交并推送**：
   ```bash
   git add .
   git commit -m "add: 描述你提交的内容"
   git push origin main
   ```
5. **提 PR**：回到你 Fork 的仓库页面，点 **Contribute → Open pull request**，填写说明后提交

维护人 review 通过后会合并到主仓库。

---

### 方式 B：GitHub 网页直接编辑（改几个字、传个文件）

不用装任何软件：

1. 打开仓库里的某个文件
2. 点右上角的 **铅笔图标 ✏️**
3. 修改内容，底部填写修改说明
4. 点 **Propose changes**，GitHub 会自动帮你 Fork 并提 PR

> 适合：小修小补、临时传个文件

---

### 方式 C：GitHub Desktop（不想记命令）

1. 下载安装 [GitHub Desktop](https://desktop.github.com/)
2. `File → Clone repository`，URL 填 `https://github.com/你的用户名/Embedded-Main`
3. 在本地改文件 → Desktop 自动检测变化
4. 填 Summary → `Commit to main` → `Push origin`
5. 打开浏览器提 PR

> 适合：不喜欢命令行的同学

---

### 方式 D：直接发给维护人（最省事，完全不会 Git）

如果你以上都不想学：

- 把文件整理好，通过 **QQ / 微信 / 邮箱 / 网盘** 直接发给维护人
- 维护人帮你检查格式、放到正确位置、提交到仓库

> 适合：偶尔贡献一次、或者文件很大/格式特殊的情况

## 六、提交要求

不强制规定文档格式，但请满足以下基本要求：

1. **文件放到正确的分类文件夹里**（见 README 的仓库内容目录）
2. **文件名清晰**，不要叫 `新建文本文档.txt` 或 `document (1).pdf`
3. **Markdown 优先**：能用 .md 尽量用 .md，Git 友好、方便修改
4. **不要传大文件**：视频、安装包等超过 10MB 的请放网盘链接
5. **不要传个人隐私信息**：学号、身份证号、手机号等请打码

**属于分类目录外的文件，存放在文件夹中，文件夹命名规范:内容描述-提交人姓名，内容不同放不同文件夹**
