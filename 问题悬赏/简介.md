# 问题悬赏

> 有问题？有技巧？发出来，大家一起进步。

本文件夹是仓库的**双向需求与分享入口**，不是知识付费，而是学习互助。

## 双向文档

- **方向 A：提问** — 不会的同学提出想学习的技术点或资料需求
- **方向 B：分享** — 学长学姐推荐好用的工具、技巧，或提出值得整理的知识点

## 如何使用

1. **提问题/分享技巧**：直接在 GitHub Issues 里新建一个 issue
2. **解答问题**：在 issue 下回复讨论，或整理后提交 PR
3. **提交规范**：解决某个 issue 的 PR 或 commit，标题写 `solve#issue号`
4. **关闭问题**：维护人合并 PR 后关闭对应 issue

👉 [点击前往 Issues](https://github.com/dev-change/Embedded-Main/issues)

## 提交激励

针对 issue 中的问题，或自己平时整理的笔记、项目代码、八股面经等内容，提交到仓库后：

- 维护人合并 → 评审人评分 → 每周评选 **3~4 名优秀提交**
- 奖励 **20~30 元**，覆盖部分 AI token 费用

> 详细评选准则待公布

## 管理规则

- 所有问题统一在 [Issues](https://github.com/dev-change/Embedded-Main/issues) 中管理
- 每周一维护人检查已关闭的 issues，确认关联的 PR 已合并
- 未解决的 issues 持续开放，等待解答
