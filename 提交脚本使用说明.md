# 提交脚本使用说明

> 一键提交资料到 Embedded-Main 仓库，省去记命令的麻烦。

---

## 使用方法

### 1. 前提条件

- 已安装 [Git](https://git-scm.com/download/win)
- 已 Fork 仓库（如果是普通成员）
- 已克隆仓库到本地

### 2. 运行脚本

1. 把 `submit.bat` 放到你的 `Embedded-Main` 文件夹里
2. 双击运行 `submit.bat`
3. 按提示操作即可

### 3. 操作流程

```
[Step 1/5] Syncing with upstream...
[Step 2/5] Enter commit message...
[Step 3/5] Adding files...
[Step 4/5] Creating commit...
[Step 5/5] Pushing to your Fork...
```

### 4. 创建 PR

脚本推送完成后，会提示你访问链接创建 Pull Request：

```
https://github.com/dev-change/Embedded-Main/compare
```

点进去，填写 PR 标题和说明，提交即可。

---

## 功能特点

| 功能 | 说明 |
|------|------|
| 自动检测更改 | 没有更改时会提示退出 |
| 自动配置上游 | 第一次使用会自动添加 upstream |
| 自动同步 | 检测到落后上游时提示同步 |
| 双模式支持 | 支持 Fork+PR 和直接 push |
| 错误提示 | 出错时给出具体原因和修复建议 |

---

## 常见问题

**Q: 提示"当前目录不是 git 仓库"？**  
A: 请把脚本放到 `Embedded-Main` 文件夹里再运行。

**Q: 推送失败？**  
A: 检查是否已 Fork 仓库，或是否配置了 GitHub 认证。

**Q: 可以修改脚本吗？**  
A: 可以，脚本就是普通的批处理文件，用记事本打开就能改。
